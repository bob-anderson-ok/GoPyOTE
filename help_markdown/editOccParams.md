# Edit/Enter Occultation Parameters

The **Edit/Enter Occultation Parameters** dialog is used to create and edit the JSON5 parameter files that drive the IOTAdiffraction external diffraction-image generator. It is opened by clicking the **Edit/Enter Occultation Parameters** button at the bottom of the main window, or automatically after computing shadow velocities from the **Process OWC occelmnt file** dialog.

##

## Buttons

- **Browse** — Opens a file dialog to select and load an existing `.occparams` parameters file. All fields are populated from the file. The file path is remembered for future sessions. Only `.occparams` files are shown in the file dialog.

- **Write** — Opens a file save dialog to write the current field values to a parameters file. See **Writing a Parameters File** below for important details.

- **Cancel** — Closes the dialog. If any field has been modified since the last load or save, a confirmation prompt appears warning about unsaved changes.

##

## Writing a Parameters File

When you click **Write**, a file save dialog opens in the `OCCULTATION-PARAMETERS` directory (created automatically if it does not exist). There are two important rules enforced by GoPyOTE:

1. **You must provide your own file name.** When the dialog is first opened after using the **Process OWC occelmnt file** workflow, the parameters shown are loaded from a temporary file called `from_occelmnt`. This file is auto-generated by GoPyOTE and is overwritten every time you run the Calculate step. You cannot save back to `from_occelmnt` — if you attempt to, you will receive an error. Instead, you must choose a meaningful name of your own (e.g., `Myrtus_2025Feb22`). This ensures that your reviewed and edited parameters are preserved separately from the temporary working file.

2. **The `.occparams` extension is enforced.** Regardless of what extension you type (or omit), GoPyOTE will strip it and replace it with `.occparams`. For example, if you enter `Myrtus_2025Feb22.txt`, the file will be saved as `Myrtus_2025Feb22.occparams`. This ensures that all parameter files are consistently named and can be found by the Browse dialog, which filters for `.occparams` files only.

After a successful save, the dialog closes automatically and the saved state is considered clean (no unsaved-changes warning on next Cancel).

##

## Auto-Load Behavior

When the dialog opens, it automatically loads the most recently used parameters file (persisted across sessions). This means you can close and reopen the dialog without losing your last set of parameters.

The **Process OWC occelmnt file** dialog also creates a parameters file (named `from_occelmnt`) and sets it as the auto-load target before opening this dialog, so computed values appear pre-filled. When opened this way, a **Fresnel Scale** popup is also displayed showing the Fresnel scale, samples per Fresnel scale, and guidance on minimum resolution needed for diffraction analysis.

##

## File Format

Parameters files use JSON5 format (with `.occparams` extension), which is JSON with relaxed syntax (trailing commas allowed, comments supported). A typical file looks like:

```json5
{
  "title": "(9203) Myrtus 2025 Feb 22",
  "window_size_pixels": 600,
  "fundamental_plane_width_km": 40,
  "fundamental_plane_width_num_points": 2000,
  "distance_au": 2.33,
  "dX_km_per_sec": 5.074,
  "dY_km_per_sec": -0.904,
  "observation_wavelength_nm": 550,
  "percent_mag_drop": 75,
  "exposure_time_secs": 0.2,
  "limb_darkening_coeff": 0.7,
  "path_to_qe_table_file": "qhy174m.qe",
  "star_class": "K",
  "main_body": {
    "x_center_km": 5.8,
    "y_center_km": 0.6,
    "major_axis_km": 17.6,
    "minor_axis_km": 8,
    "major_axis_pa_degrees": 98.3,
  },
  "satellite": {
    "major_axis_km": 7.5,
    "minor_axis_km": 2.8,
    "major_axis_pa_degrees": 94,
    "x_center_km": -5.8,
    "y_center_km": -0.6,
  },
}
```

##

## Usage with IOTAdiffraction

After editing and saving a parameters file, use the **Run IOTAdiffraction** button to generate diffraction images. The Run IOTAdiffraction button opens the `OCCULTATION-PARAMETERS` directory and shows only `.occparams` files for selection. Upon completion, the generated diffraction image and title are displayed as a splash overlay in the main window.

**Name of camera QE file** — if this field contains a filename (e.g., `qhy174m.qe`), GoPyOTE automatically prefixes it with `CAMERA-QE/` when running IOTAdiffraction, so the file is resolved as `CAMERA-QE/qhy174m.qe` relative to the application directory. The original parameters file is not modified.
