# Fit Page — Procedures and Usage

## 

## Overview

The Fit page provides tools for fitting a theoretical diffraction light curve to an observed occultation light curve. It uses Normalized Cross-Correlation (NCC) to find the best time alignment between the two curves by searching across a range of observation path offsets. A Monte Carlo procedure estimates the uncertainty in the derived edge times. A Noise-Induced-Event (NIE) analysis provides a plot that can guide the observer in deciding whether the *event* is sufficently to the right of the noise-induced-drops histogram to be reported.

## 

## Prerequisites

## 

Before running a fit, the following must be in place:

## 

1. **A single light curve displayed** — exactly one light curve column must be selected for display.
2. **Light curve scaled to unity** — the baseline must be normalized (via the Calculate Baseline step).
3. **Parameters file loaded** — a parameters file (`.occparams`) describing the occultation geometry must be present. This can come from an immediate generation or from one saved in the **-RESULTS** folder from a previous run)
4. **Diffraction image available** — `targetImage16bit.png` must exist in the working directory (generated by IOTAdiffraction.exe) as well as `geometricImage.png`. These can come from an immediate generation or from one saved in the **-RESULTS** folder from a previous run.

## 

## Step-by-Step Procedure

## 

### 1. Define Baseline Regions

On the interactive plot, click pairs of points to mark baseline regions — segments of the light curve where the star is unocculted.

## 

- Click two points to define one baseline region (a pair).
- Repeat to add more baseline regions for better statistics.
- Click on a previously marked point to remove that pair.
- Click **Clear marked points** (above the plot) to remove all selected and marked points at once.

## 

### 2. Normalize Baseline

## 

Click **Normalize baseline and estimate noise sigma (used for Monte Carlo and NIE trials)** to:

## 

1. **Compute the mean** of all data points within the selected baseline regions.
2. **Scale to unity** — divide all light curve values by the baseline mean so the unocculted level equals 1.0.
3. **Estimate noise sigma** — for each baseline point, compute `(value / mean) - 1.0`. Fit a Gaussian to these residuals and record the standard deviation (sigma), which is used in Monte Carlo and NIE uncertainty estimation.
4. **Display a noise histogram** (only if **Show diagnostics plots** is checked) — shows the distribution of baseline noise with a Gaussian fit overlay, reporting the mean and sigma.
5. The lag correlation coefficients of the observation are computed during this process, so include as many points as possible at this step, before any trimming has been applied.

## 

After this step, the status bar shows the baseline mean, number of points used, and the estimated noise sigma.

## 

### 3. Search range for observation path offset

## 

**Intial offset** and **Final offset** are prefilled and set the observation path offset search to the maximum possible need to move the observation path from the asteroid center to the edge of the asteroid.

## 

### 4. Configure Path Offset Search (Optional)

## 

- **Initial offset** — starting path offset in km.
- **Final offset** — ending path offset in km.
- **Number of steps** — auto-calculated when both offsets are entered. The step size equals the diffraction image pixel resolution (`FundamentalPlaneWidthKm / FundamentalPlaneWidthNumPoints`), ensuring full-resolution sampling.

## 

When both offset entries are filled and focus leaves the entry, a **preview image** is shown: the 8-bit diffraction image with observation paths drawn at the start and end offsets, so the search range can be visually verified.

## 

## 

### 5. Run the Fit

## 

Click **Fit** to start. An **Abort** button appears to the right of the Fit button while the search is running. The procedure:

## 

1. **Loads parameters** from the selected parameters file.
2. **Generates a theoretical light curve** by extracting the diffraction profile along the observation path through the 16-bit diffraction image, converting distance to time using the shadow speed (`sqrt(dX^2 + dY^2)`), and applying camera exposure integration if `exposure_time_secs` is set.
3. **Detects geometric shadow edges** from the geometric shadow image.
4. **Runs an NCC sliding fit** — slides the theoretical curve across the observed curve in steps of one frame period, computing the NCC at each offset. The NCC is weighted by the overlap fraction (number of overlapping points / maximum overlap) to penalize partial overlaps.
5. **Finds the best alignment** — the time offset with the highest weighted NCC.

## 

## 

**Aborting a fit search:** If a path offset search is taking too long, click the **Abort** button. The search stops after the current step is completed, and results are displayed using whatever steps have been completed so far. The best path offset is selected from the completed steps. This is useful when searching a large range, and you can see that the region of interest has already been covered.

## 

#### Fit Results Displayed

## 

- **NCC Fit Result** window (only if **Show diagnostics plots** is checked) — plot of NCC (weighted and unweighted) versus time offset, with the peak marked.
- **Fit Result** window — overlay plot of observed (blue) and theoretical (light red) light curves at the best alignment, with sampled theoretical points (dark red dots) and geometric shadow edges (green dashed vertical lines).
- **Observation Path** window — the diffraction image with the observation path drawn at the best-fit offset.
- **Fit Edge Times** dialog — reports NCC value, time offset, path offset, edge times in timestamp format (hh:mm:ss.ssss), event duration, number of samples between event edges, and the minimum theoretical value at the event.

## 

All fit results are logged to the session log file.

## 

## Monte Carlo Uncertainty Estimation

## 

After a successful fit, the Monte Carlo procedure estimates the uncertainty in edge times. **This should always be run before a report is submitted.**

## 

**It is recommended that you trim closely around the event to speed up the Monte Carlo process.**

## 

### GUI items

## 

- **Monte Carlo trials** — entry box for the number of trials (default: 1000).
- **Narrow MC offset search (+/-) 20 steps** defaults to checked. This restriction speeds up the Monte Carlo process as it is not necessary or useful to try every possible lightcurve from the asteroid center to the edge. A narrow band of theoretical lightcurves around the 'best' fit is enough.
- **Enable manual selection of points for NIE analysis** — checkbox (default: unchecked). When checked, the NIE window and event drop are determined by points you click on the light curve rather than from the fit edge times.

**Show diagnostics plots** — checkbox in the Setting tab (default: unchecked). When checked:

- The **Baseline Noise Histogram** is displayed after normalizing the baseline.
- The **NCC Fit Result** window is displayed after each fit.
- The **Peak NCC vs Path Offset** plot is displayed after a path offset search.
- The results dialog includes a scrollable list of individual trial edge times, durations, and path offsets (up to 100 trials displayed).
- Histogram windows with Gaussian fits are displayed for each edge time and for the event duration.

## 

### Procedure

## 

Click **Run Monte Carlo** to start. An **Abort** button appears to the right of the Run Monte Carlo button while trials are running. The procedure:

## 

1. **Uses candidate theoretical curves from the fit** — the Monte Carlo procedure reuses the theoretical light curves computed during the fit, or, more commonly, a range of theoretical lights surrounding the 'best' light curve. 

2. **Runs N trials**, each consisting of:
   
   a. **Create a noisy observation** — take the sampled theoretical curve from the best fit and add correlated Gaussian noise drawn from N(0, noiseSigma), where noiseSigma is the baseline noise sigma estimated during the Normalize Baseline step.
   
   b. **Search across candidates** — run the NCC sliding fit against each candidate curve and select the one with the highest NCC.
   
   c. **Record results** — store the edge times and path offset for this trial.

3. **Computes statistics** — mean and standard deviation (3 sigma) of edge times and event duration across all completed trials.

## 

**Aborting Monte Carlo:** If the trials are taking too long, click the **Abort** button. The Monte Carlo run stops after the current trial completes. The report is then generated based on however many trials were completed, rather than the originally requested number. The results dialog, histograms, and final report all reflect the actual number of completed trials. This is useful for getting a quick preliminary estimate of uncertainty — even a few hundred trials can give a reasonable estimation of the edge time uncertainty.

## 

### Results

## 

- **Monte Carlo Edge Time Uncertainty** dialog — shows the best-fit edge times with 3-sigma uncertainties, and the event duration with its 3-sigma uncertainty, along with the number of completed trials. Example format:
  
  ## 
  
  ```
  Edge 1: 03:58:34.5678 +/- 0.0123 sec (3 sigma)
  Edge 2: 03:58:35.6789 +/- 0.0145 sec (3 sigma)
  
  Duration: 1.1111 +/- 0.0190 sec (3 sigma)
  ```
