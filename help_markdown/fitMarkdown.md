# Fit Page — Procedures and Usage

## 

## Overview

The Fit page provides tools for fitting a theoretical diffraction light curve to an observed occultation light curve. It uses Normalized Cross-Correlation (NCC) to find the best time alignment between the two curves, optionally searching across a range of observation path offsets. A Monte Carlo procedure estimates the uncertainty in the derived edge times.

## 

## Prerequisites

Before running a fit, the following must be in place:

1. **A single light curve displayed** — exactly one light curve column must be selected for display.
2. **Light curve scaled to unity** — the baseline must be normalized (via the Calculate Baseline step).
3. **Parameters file loaded** — a JSON5 parameters file describing the occultation geometry must be loaded (File menu or auto-load).
4. **Diffraction image available** — `targetImage16bit.png` must exist in the working directory (generated by IOTAdiffraction.exe).

## 

## Step-by-Step Procedure

## 

### 1. Define Baseline Regions

On the interactive plot, click pairs of points to mark baseline regions — segments of the light curve where the star is unocculted.

- Click two points to define one baseline region (a pair).
- Repeat to add more baseline regions for better statistics.
- Click on a previously marked point to remove that pair.

## 

### 2. Calculate Baseline

Click **Calculate Baseline mean** to:

1. **Compute the mean** of all data points within the selected baseline regions.
2. **Extract noise** — for each baseline point, compute `(value / mean) - 1.0`. These residuals are stored for later use in Monte Carlo uncertainty estimation.
3. **Scale to unity** — divide all light curve values by the baseline mean so the unocculted level equals 1.0.
4. **Display a noise histogram** — shows the distribution of baseline noise with a Gaussian fit overlay, reporting the mean and sigma.

After this step, the status bar shows the baseline mean, number of points used, and number of noise samples extracted.

## 

### 3. Configure Path Offset

**Path Perpendicular Offset (km)** — an optional entry that overrides the path offset from the parameters file. If left blank, the value from the parameters file is used.

## 

### 4. Configure Path Offset Search (Optional)

The **Search range for observation path offset** card allows fitting across multiple path offsets to find the best one:

- **Initial offset** — starting path offset in km.
- **Final offset** — ending path offset in km.
- **Number of steps** — auto-calculated when both offsets are entered. The step size equals the diffraction image pixel resolution (`FundamentalPlaneWidthKm / FundamentalPlaneWidthNumPoints`), ensuring full-resolution sampling.

When both offset entries are filled and focus leaves the entry, a **preview image** is shown: the 8-bit diffraction image with observation paths drawn at the start and end offsets, so the search range can be visually verified.

If the search range fields are left empty, a single fit at the configured path offset is performed instead.

##

### 5. Run the Fit

Click **Fit** to start. The procedure:

1. **Loads parameters** from the selected parameters file.
2. **Generates a theoretical light curve** by extracting the diffraction profile along the observation path through the 16-bit diffraction image, converting distance to time using the shadow speed (`sqrt(dX^2 + dY^2)`), and applying camera exposure integration if `exposure_time_secs` is set.
3. **Detects geometric shadow edges** from the geometric shadow image.
4. **Runs an NCC sliding fit** — slides the theoretical curve across the observed curve in steps of one frame period, computing the NCC at each offset. The NCC is weighted by the overlap fraction (number of overlapping points / maximum overlap) to penalize partial overlaps.
5. **Finds the best alignment** — the time offset with the highest weighted NCC.

If a **search range** is specified, this process repeats for each path offset step and a **Peak NCC vs Path Offset** plot is shown. The path offset with the highest peak NCC is selected.

## 

#### Fit Results Displayed

- **NCC Fit Result** window — plot of NCC (weighted and unweighted) versus time offset, with the peak marked.
- **Fit Result** window — overlay plot of observed (blue) and theoretical (light red) light curves at the best alignment, with sampled theoretical points (dark red dots) and geometric shadow edges (green dashed vertical lines).
- **Observation Path** window — the diffraction image with the observation path drawn at the best-fit offset.
- **Fit Edge Times** dialog — reports NCC value, time offset, path offset, edge times in timestamp format (hh:mm:ss.ssss), and event duration.

All fit results are logged to the session log file.

## 

## Monte Carlo Uncertainty Estimation

After a successful fit, the Monte Carlo procedure estimates the uncertainty in edge times.

## 

### Configuration

- **Monte Carlo trials** — entry box for the number of trials (default: 1000).
- **Show individual trial results** — checkbox (default: unchecked). When checked, the results dialog includes a scrollable list of individual trial edge times, durations, and path offsets.
- **Show histograms** — checkbox (default: unchecked). When checked, histogram windows with Gaussian fits are displayed for each edge time and for the event duration.

## 

### Procedure

Click **Run Monte Carlo** to start. The procedure:

1. **Pre-computes candidate theoretical curves** — generates theoretical light curves for 51 path offsets (25 steps on each side of the best-fit offset, at pixel resolution). A progress dialog is shown during this phase.

2. **Runs N trials**, each consisting of:
   a. **Create a noisy observation** — take the sampled theoretical curve from the best fit and add noise by sampling with replacement from the extracted baseline noise (bootstrap resampling).
   b. **Search across candidates** — run the NCC sliding fit against each pre-computed candidate curve and select the one with the highest NCC.
   c. **Record results** — store the edge times and path offset for this trial.

3. **Computes statistics** — mean and standard deviation (1 sigma) of edge times and event duration across all trials.

## 

### Results

- **Monte Carlo Edge Time Uncertainty** dialog — shows the standard deviation (1 sigma) for each edge and for the duration.
- **Histograms** (if enabled) — separate windows for each edge time distribution and the duration distribution, each with a Gaussian curve fit overlay.
- **Individual trial results** (if enabled) — scrollable list showing per-trial edge times, duration, and path offset.
- ## 

### Final Report (Log File)

A final report is written to the log file combining the fit edge times (in timestamp format) with the Monte Carlo standard deviations:

```
--- Final Report ---
  NCC=0.9523, path offset=12.345 km
  Edge 1: 03:58:34.5678 +/- 0.0123 sec (1 sigma)
  Edge 2: 03:58:35.6789 +/- 0.0145 sec (1 sigma)
  Duration: 1.1111 +/- 0.0190 sec (1 sigma)
--- End Report ---
```
