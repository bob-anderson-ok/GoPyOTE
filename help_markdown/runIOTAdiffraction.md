# Run IOTAdiffraction

The **Run IOTAdiffraction** button, located at the bottom of the main window, launches the external IOTAdiffraction.exe program to generate diffraction images from an occultation parameters file.

## 

## How It Works

1. **Select a parameters file.** Clicking the button opens a file selection dialog in the `OCCULTATION-PARAMETERS` directory, showing only `.occparams` files. Choose a parameters file — typically one created or edited via the **Edit/Enter Occultation Parameters** dialog or generated by the **Process OWC occelmnt file** workflow.

2. **IOTAdiffraction.exe runs.** The external program is launched with the selected parameters file as its argument. It runs in the application directory (`appDir` ), where it reads the parameters and generates diffraction images.

3. **Live output display.** A scrollable output dialog appears showing the real-time stdout and stderr output from IOTAdiffraction.exe. Stderr lines are prefixed with `[stderr]`. The dialog auto-scrolls to show the latest output.

4. **On successful completion:**
   
   - The output dialog displays `[Process completed successfully]`.
   - The main window's startup overlay is refreshed to show the newly generated diffraction image (`diffractionImage8bit.png` ) along with the event title extracted from the parameters file.
   - The parameters file path and title are persisted in preferences for future sessions.

5. **On failure:** The output dialog displays the error message from the process.

## 

## Prerequisites

- **IOTAdiffraction.exe** must be located in the same directory as the GoPyOTE executable. If it is not found, an informational dialog is shown with the expected path.

- A valid **parameters file** in JSON5 format is required. See the **Edit/Enter Occ Params** help topic for the file format and field descriptions.

## 

## Generated Files

IOTAdiffraction.exe produces the following files in the application directory:

## 

## diffractionImage8bit.png

8-bit diffraction image used for the startup splash display and observation path overlays. It has auto-contrast for easy viewing and is not used for 'science'.

## targetImage16bit.png

16-bit diffraction image used for light curve extraction and fitting. No auto-contrast so values are correct for 'science'.

## geometricShadow.png

Geometric shadow image. Used to define physical edges of objects.

## 

## Typical Workflow

1. Create or load a parameters file using **Edit/Enter Occultation Parameters** (or generate one from an OWC occelmnt file).
2. Click **Run IOTAdiffraction** and select the parameters file.
3. Wait for the process to complete. The diffraction image splash confirms success.
4. Load a light curve CSV and use the Fit tab to align the observed light curve against the generated diffraction pattern.

## 

## Persistence

- The path to the last-used parameters file is saved in preferences and used to populate the startup overlay on subsequent launches.
- The title from the parameters file is also persisted and displayed in the main window title bar and plot titles.
